# Сборка  

make из корня проекта    

# Запуск   

. /my_program  

# Параметры  

Параметры ( номера портов, размер буфера) задаются в файле config.hpp  

# Принцип работы  

Gриложение-сервер на C/C++ для GNU/Linux, который асинхронно 
обрабатывает клиентские подключения по протоколам TCP и UDP с помощью 
механизма мультиплексирования (на базе epoll) без использования сторонних 
библиотек.   

Если сообщение клиента начинается с символа /, то оно интерпретируется как 
команда.   

В противном случае зеркалируется клиенту.   

Сервер должен поддерживать следующие команды:   

/time — возврат текущего времени и даты в формате "2025-11-10 17:28:45";   

/stats — возврат статистики (общее количество подключившихся клиентов и подключенных в данный момент);   

/shutdown — завершение работы.
